Relational Data

Relational data is organized in structured tables with rows and columns, allowing relationships to be established between different tables based on shared data, commonly managed through SQL.

Relation Database

A relational database organizes data into tables with rows and columns, using SQL for data management. It allows efficient storage, retrieval, and manipulation of interconnected data.

Relational Database Model

The relational data model is a way of structuring data in tables with rows and columns, and establishing relationships between these tables based on shared data, facilitating efficient data management and retrieval. It is a fundamental concept in relational database management systems (RDBMS).

Components of Table

Columns (Attributes) : Columns represent the individual data fields in the table, defining the type of data they can hold.

Rows (Records) : Rows are the individual entries in the table, containing data related to specific entities or records.

Cells : Cells are the intersections of rows and columns, holding single values representing specific data.

Primary Key : A primary key is a unique identifier for each row, ensuring that each record can be uniquely identified and used for establishing relationships with other tables.

Keys

Keys refer to specific columns or combinations of columns that play essential roles in defining the structure and relationships within a database.

Here are some common types of keys :

Primary Key : A primary key is a unique identifier for each row in a table. It ensures that each record can be uniquely identified and serves as a reference for establishing relationships with other tables.

Foregin Key : A foreign key is a column or set of columns in one table that refers to the primary key of another table. It creates relationships between tables, enabling data integrity and data retrieval across related tables.

Candidate Key : A candidate key is a column or combination of columns that could potentially serve as a primary key. It must be unique and capable of uniquely identifying each row in the table.

Super Key : A super key is a set of one or more columns that, taken together, uniquely identify rows in a table. It may or may not be minimal in size, meaning it may contain more columns than needed to identify rows uniquely.

Composite Key : A composite key is a key that consists of multiple columns used together to uniquely identify rows in a table. It is an alternative to using a single column as the primary key..

These keys play a crucial role in establishing relationships between tables and ensuring data integrity within a relational database. They are vital components for data retrieval, modification, and maintenance in a structured and efficient manner.

Classification of SQL Statements (for 12th grade)

DML (Data Manipulation Language) : Used to manipulate data in the database, such as INSERT, UPDATE, and DELETE statements.

DDL (Data Definition Language) : Used to define and manage the database structure, including SELECT, CREATE, ALTER, and DROP statements.
