Create table command

This query is used to create a table in the selected database.

Create table <table-name>
(<column_name> <data_type>,
<column_name> <data_type>,
<column_name> <data_type>);

SQL Constraints

SQL constraints are the rules or checks enforced on the data columns of a table.

NOT NULL

It will create a table with NOT NULL constraint to its first column.

Create table <table_name>
( <col1> <data_type> NOT NULL,
<col2> <data_type>,
<col3> <data_type>);

DEFAULT

DEFAULT constraint provides a default value to a column.

Create table <table_name>
( <col1> <data_type> DEFAULT 50,
<col2> <data_type>,
<col3> <data_type>);

CHECK

CHECK constraint ensures that all values in a column satisfy certain conditions.

Create table <table_name>
( <col1> <data_type> CHECK (condition),
<col2> <data_type>,
<col3> <data_type>);

Keys

In SQL, keys are used to uniquely identify records in a database table and establish relationships between different tables. There are several types of keys, each serving a different purpose

Primary Key

A primary key is a column or a combination of columns in a table that uniquely identifies each row in the table. It must have unique values and it cannot contain NULL values. A table can have only one primary key.

Create table <table_name>
( <col1> <data_type> Primary Key,
<col2> <data_type>,
<col3> <data_type>);

Foreign Key

A foreign key establishes a link or relationship between two tables based on the values in one or more columns. It points to the primary key or a unique key in another table, thereby enforcing referential integrity. It helps maintain data consistency and ensures that the data between related tables remains synchronized.

CREATE TABLE <table_name> (
<col1 <data_type>,
<col2 <data_type>,
<col3 <data_type>,
...
FOREIGN KEY (<column_name>) REFERENCES <parent_table> (<parent_column>) );

Unique

A unique key is similar to a primary key in that it enforces uniqueness for a column or a combination of columns in a table. Unlike a primary key, a unique key can contain NULL values, but if it contains values, those values must be unique.

Create table <table_name>
( <col1> <data_type> UNIQUE,
<col2> <data_type>,
<col3> <data_type>);
