File Handling in Python

In Python, file handling refers to the process of working with files, which can include reading from and writing to files. File handling is essential for various tasks like data storage, data retrieval, configuration management, and more. Python provides built-in functions and methods for file handling, making it easy to work with files.

Here are some common file handling operations in Python:

Opening a File

You can use the `open()` function to open a file. It takes two arguments: the file name and the mode in which you want to open the file (e.g., read, write, append, binary, etc.)

# Open a file for reading
file = open("example.txt", "r")


# Open a file for writing (creates a new file if it doesn't exist)
file = open("example.txt", "w")


# Open a file for appending (adds data to the end of the file)
file = open("example.txt", "a")

Reading from a File

You can use various methods like `read()`, `readline()`, or `readlines()` to read data from a file.

# Reading the entire file
data = file.read()


# Reading a single line
line = file.readline()


# Reading all lines into a list
lines = file.readlines()

Writing to a File

You can use the `write()` method to write data to a file.

file.write("Hello, World!")

Closing a File

After working with a file, it's essential to close it using the close() method to release system resources.

file.close()

Using a Context Manager

It's a good practice to use the `with` statement when working with files. It automatically closes the file when you're done with it, ensuring that resources are properly managed.

File Modes

`"r"` (Read Mode):

1] Opens the file for reading.
2] If the file doesn't exist, it raises a `FileNotFoundError`.
3] Default mode if not specified explicitly.
Example: `open("example.txt", "r")`

`"w"` (Write Mode):

1] Opens the file for writing.
2] If the file exists, it truncates (clears) its content.
3] If the file doesn't exist, it creates a new file.
Example: `open("example.txt", "w")`

`"a"` (Append Mode):

1] Opens the file for writing, but appends data to the end of the file.
2] If the file doesn't exist, it creates a new file.
Example: open("example.txt", "a")

`"x"` (Exclusive Creation Mode):

1] Creates a new file.
2] If the file already exists, it raises a `FileExistsError`.
Example: `open("new_file.txt", "x")`

These modes are used when opening files with the open() function in Python. They determine how the file can be accessed and what operations can be performed on it. It's essential to choose the appropriate mode for your specific file handling needs to ensure data integrity and security.

2 in one mode

Python provides three additional file modes: `"w+"`, `"r+"`, and `"a+"`. These modes combine both reading and writing operations, allowing you to both read from and write to a file. Here's a brief explanation of each:

`"w+"` (Write and Read Mode):

1] Opens the file for both reading and writing.
2] If the file exists, it truncates (clears) its content.
3] If the file doesn't exist, it creates a new file.
4] You can use `read()` and `write()` methods to perform both read and write operations.
Example: `open("example.txt", "w+")`

`"r+"` (Read and Write Mode):

1] Opens the file for both reading and writing.
2] The file must already exist; otherwise, it raises a FileNotFoundError.
3] You can use read() and write() methods to perform both read and write operations.
Example: `open("example.txt", "r+")`

`"a+"` (Append and Read Mode):

1] Opens the file for both reading and writing, with data appended to the end of the file.
2] If the file doesn't exist, it creates a new file.
3] You can use `read()` and `write()` methods to perform both read and write operations.
Example: `open("example.txt", "a+")`

These modes are useful when you need to perform a combination of read and write operations on the same file. However, you should be cautious when using them to avoid unintended consequences, such as overwriting or losing data in the file. Always ensure that your code handles file operations correctly to maintain data integrity.
